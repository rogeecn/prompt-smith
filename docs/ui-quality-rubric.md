# UI 质量评估规则（Rubric）

本规则用于评估 Prompt Smith 的 UI 质量，作为发布前检查与迭代对比依据。
总分 100，建议每次发布前按页面/功能路径逐项打分。

## A. 结构与信息层级 (20)

- 信息层级清晰：标题/内容/操作按钮层次分明，用户能在 3 秒内理解当前步骤 (0-6)
- 关键路径清晰：核心任务路径无干扰或强干扰提示 (0-8)
- 版式一致性：布局、间距、模块顺序在页面间一致 (0-6)

## B. 可读性与视觉噪声 (20)

- 行高、字级与对比度符合阅读舒适度 (0-6)
- 视觉噪声控制：无多余线框、阴影、标签泛滥 (0-6)
- 长内容承载：长文本不会撑破布局，滚动在合理区域 (0-8)

## C. 交互反馈与状态表达 (20)

- Loading/思考状态明确：用户能在 2 秒内判断是否在处理 (0-8)
- 表单/问答流程清晰：提交后状态变化符合预期 (0-6)
- 错误提示可行动：提示包含明确下一步 (0-6)

## D. 功能可达性与效率 (15)

- 核心功能入口清晰（新建、继续对话、导出制品）(0-6)
- 常用操作不超过 3 步 (0-5)
- 键盘/快捷行为可用（Enter 发送、Shift 换行）(0-4)

## E. 一致性与语义准确性 (15)

- 文案一致且语义明确 (0-6)
- 状态/按钮文案与动作一致 (0-5)
- 模块命名与功能一致（如“制品”与“导出”）(0-4)

## F. 移动端适配 (10)

- 移动端无遮挡、抽屉可用、内容不溢出 (0-6)
- 触控目标大小、可滚动区域合理 (0-4)

## 评分结果解释

- 90–100：优秀，产品级稳定体验
- 75–89：良好，有小瑕疵但可上量
- 60–74：可用但需优化
- <60：交互明显阻碍使用

## 执行方式

1. 每个页面走一遍“主流程 + 异常流程”，逐项打分。
2. 记录不达标项，定位到具体组件/路径。
3. 每次迭代对比得分变化，做趋势跟踪。
